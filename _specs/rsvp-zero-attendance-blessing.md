# Feature Spec: RSVP 零出席人数 — 仅送祝福模式

## Overview

当宾客无法亲身出席婚礼，但仍希望提交表单送上祝福时，当前表单的出席人数最少为 1，无法表达"不出席但送祝福"的意愿。本功能允许宾客将出席人数设为 0，并在此状态下调整表单文案与提交后的反馈，让无法到场的宾客也能优雅地留下祝福。

---

## Goals

- 允许出席人数减少至 0
- 出席人数为 0 时，自动切换为"仅送祝福"状态，调整按钮文案与成功提示
- 提交后邮件通知中注明该宾客为"无法出席，仅送祝福"
- 保持现有出席流程不受影响

---

## Non-Goals

- 不新增独立的"无法出席"入口页或独立表单
- 不需要区分"明确拒绝"与"送祝福"两种状态（统一为人数=0即送祝福）
- 不需要在页面上额外展示已送祝福的宾客列表

---

## User Stories

**作为无法出席的宾客**，我希望能在 RSVP 表单中将人数减至 0，并提交我的祝福留言，让新人知道我虽然不能到场，但心意已到。

**作为新人（网站所有者）**，当我收到邮件通知时，我希望能清晰看出哪位宾客是"无法出席仅送祝福"，方便区分统计。

---

## Functional Requirements

### 出席人数控件

| 当前行为 | 目标行为 |
|----------|----------|
| 最小值为 1，减号在 count=1 时禁用/无效 | 最小值改为 0，允许减至 0 |
| 始终显示"位" | count=0 时显示提示文字，例如"无法出席" |

### 表单文案联动（count = 0 时）

| 元素 | count ≥ 1 | count = 0 |
|------|-----------|-----------|
| 出席人数区域副文字 | "位" | "无法出席" 或类似柔和表达 |
| 提交按钮文案 | "确 认 出 席" | "送 上 祝 福" |
| 祝福留言 label 提示 | （选填） | （选填，欢迎留下心意） 或类似引导文字 |

### 成功页面联动

| 情况 | 成功文案 |
|------|----------|
| count ≥ 1 | 现有文案："已收到您的回复 / 感谢 {name}，期待与您相见" |
| count = 0 | 替换为："已收到您的祝福 / 感谢 {name}，您的心意我们已收到" 或类似温暖文案 |

### 邮件通知（联动现有 RSVP 邮件功能）

- count = 0 时，邮件主题改为：`[婚礼回执] {name} 送上祝福（无法出席）`
- 邮件正文中出席人数显示为"0（无法出席，仅送祝福）"

---

## Open Questions

1. **count = 0 时"无法出席"文字的视觉样式**：是否需要与 count ≥ 1 时的"位"在颜色/字重上有所区分，以作视觉提示？建议：
   - A. 仅替换文字，样式不变
   - B. 文字颜色切换为 `--color-ink-ghost`（更淡），以柔和表达
   - C. 在人数控件旁增加一个小图标或标签

2. **祝福留言在 count = 0 时是否变为必填**？建议：
   - A. 保持选填，不强制
   - B. 改为必填，确保无法出席的宾客留下文字祝福

---

## Acceptance Criteria

- [ ] 出席人数可以减少至 0，减号在 count=0 时才不可再减
- [ ] count=0 时，提交按钮显示"送 上 祝 福"而非"确 认 出 席"
- [ ] count=0 时，成功页面显示"已收到您的祝福"相关文案，而非"期待与您相见"
- [ ] count=0 时，邮件通知内容体现"无法出席，仅送祝福"
- [ ] count ≥ 1 的现有流程完全不受影响
