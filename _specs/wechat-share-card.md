# Feature Spec: 微信分享卡片（Open Graph / 微信分享卡）

## Overview

当用户将网站链接分享到微信时，聊天窗口中显示的不是裸链接，而是类似微信公众号文章的小卡片——带有封面图片、标题和摘要。卡片内容可由网站所有者在代码或配置中自定义。

---

## Goals

- 在微信内分享链接时，自动展示图文卡片（含封面图、标题、摘要）
- 网站所有者可以方便地配置卡片内容（图片、标题、摘要）
- 不同页面可以有各自的卡片配置（首页与 RSVP 页等可以不同）

---

## Non-Goals

- 不需要微信公众号授权或 JS-SDK 接入（仅使用 Open Graph meta 标签即可满足基本需求）
- 不需要动态生成卡片图片
- 不需要微信分享计数、追踪分析
- 不需要为微信小程序适配

---

## User Stories

**作为新人（网站所有者）**，当我把婚礼邀请网站链接发给朋友时，我希望朋友在微信聊天中看到一张好看的小卡片，而不是一串裸链接，以提升仪式感和点击意愿。

**作为新人**，我希望能够自己配置卡片上显示的图片、标题和摘要，不需要改动大量代码。

**作为被邀请的朋友**，当我在微信中看到这个卡片时，能第一眼了解这是什么，并愿意点开查看。

---

## Functional Requirements

### 卡片内容字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 标题（title） | 卡片大标题 | 「林阳阳 & XXX 的婚礼」 |
| 摘要（description） | 卡片副标题/描述文字 | 「诚挚邀请您出席我们的婚礼典礼」 |
| 封面图（image） | 卡片左侧/顶部缩略图，建议正方形 300×300px 以上 | 婚纱照或婚礼主视觉 |
| 网站名称（site_name） | 来源标识 | 「林阳阳的婚礼」 |

### 技术实现方式

- 通过在页面 `<head>` 中写入标准 Open Graph meta 标签来实现
- 微信使用 `og:title`、`og:description`、`og:image`、`og:url` 渲染卡片
- 图片需使用绝对 URL（含域名），且需公开可访问

### 配置方式

- 全局默认配置：在一处集中文件/常量中定义站点级别的默认卡片内容
- 页面级覆盖：各页面可独立指定不同的标题、摘要、图片
- 配置项变更后，重新部署即可生效，无需运行时管理界面

### 封面图托管

- 封面图可放置在项目的 `public/` 目录下（Next.js 静态资源）
- 需通过可公开访问的绝对 URL 引用（如 `https://yourdomain.com/og-image.jpg`）

---

## Technical Notes

- 当前项目为 Next.js，可使用 `next/head` 或 App Router 的 `metadata` API 注入 meta 标签
- 微信对图片的建议尺寸：正方形，至少 300×300px，文件大小建议 <500KB
- 微信会在首次分享时抓取并缓存卡片内容，更换图片后缓存可能需要时间刷新
- 如需精确控制，可加入 `twitter:card` 等兼容标签

---

## Open Questions

1. **封面图来源**：是使用现有婚纱照，还是需要设计一张专门的分享卡片主视觉？图片是否已有？
2. **多页面需求**：除首页外，RSVP 页或其他子页面是否也需要各自不同的卡片内容？
3. **域名状态**：网站是否已绑定自定义域名并可公开访问？（图片必须通过公开 URL 被微信抓取）
4. **缓存刷新**：如果卡片内容更新后想立刻看到效果，是否了解需要重新分享链接才能触发微信重新抓取？

---

## Acceptance Criteria

- [ ] 在微信中分享网站链接时，显示包含图片、标题、摘要的小卡片，而非裸链接
- [ ] 卡片标题、摘要、封面图可通过集中的配置文件修改
- [ ] 配置修改后重新部署，卡片内容随之更新
- [ ] 首页和至少一个子页面的卡片内容可独立配置
- [ ] 封面图可公开访问，微信能正常抓取到图片
