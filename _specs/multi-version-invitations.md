# Feature Spec: 多版本请柬路由

## Overview

当前婚礼邀请页面仅有一个版本（男方父母视角），部署在主域名根路径 `/`。  
实际需要支持 **4 个版本**：男方父母、女方父母、夫妻·男方场、夫妻·女方场。  
各版本共用同一套页面组件，仅文案和时间/地点数据不同，统一挂载在 `/invite/` 前缀下。

根路径 `/` 保留现有页面内容（使用男方父母数据），不做跳转。

---

## Goals

- 同一套页面组件，通过不同路径渲染不同版本
- 4 个版本各有专属 URL，方便分别转发给不同宾客
- 统一使用 `/invite/` 前缀，不占用根路径，便于域名后续扩展其他用途
- 微信分享卡片（标题、摘要）随版本自动切换
- 页面落款签名随版本自动切换
- 时间、地点数据随「男方场 / 女方场」切换

---

## Non-Goals

- 不需要登录或权限控制
- 不需要 URL 加密或一次性链接
- 不需要 A/B 测试或动态个性化
- 不需要改变页面视觉设计，仅文案数据不同
- 不需要在 RSVP 邮件中区分场次（统一收件人）
- 不需要为不同版本配置不同的微信分享图片（统一共用）

## 技术方案（概述，不含实现细节）

### 推荐：Next.js 嵌套路由 + 配置 Map

1. **新建路由目录** `src/app/invite/[variant]/page.tsx`，接收路径参数 `variant`
2. **扩展 `wedding.ts` 配置**，新增 `variants` Map，key 为 variant 字符串，value 包含该版本完整数据（venue、time、parents/coupleName、wechat 标题/摘要）
3. **页面从配置 Map 取值**，传递给各 Section 组件；variant 不存在时返回 404
4. **`generateMetadata`** 按 variant 导出对应 Open Graph 元数据
5. **根路径 `/`** 保留现有页面，不做跳转，继续使用男方父母数据

此方案无需改动任何 UI 组件，只扩展配置层。

## User Stories

**作为新郎/新娘**，我希望有一个「夫妻版」链接转发给双方共同的朋友，签名是我们自己的名字，而不是父母。

**作为男方父母**，我希望能给亲戚转发男方场次的邀请，显示我们的名字作为邀请方。

**作为女方父母**，我希望能给亲戚转发女方场次的邀请，显示女方父母的名字。

**作为宾客**，无论收到哪个链接，都能看到正确的宴会时间、地点和签名。

## Acceptance Criteria

- [ ] `/invite/groom-family` 渲染男方父母版（招远场，落款为林洪权 · 胡洪兰）
- [ ] `/invite/bride-family` 渲染女方父母版（潍坊场，落款为王少伟 · 孙凤丽）
- [ ] `/invite/groom` 渲染夫妻版·招远场（落款为林洋洋 · 王文靖）
- [ ] `/invite/bride` 渲染夫妻版·潍坊场（落款为林洋洋 · 王文靖）
- [ ] 访问未知 variant 路径时返回 404
- [ ] 根路径 `/` 保留现有页面内容，不受影响
- [ ] 每个路径的微信分享卡片标题、摘要与版本一致
- [ ] 每个路径的 Open Graph `<title>` 与版本一致
