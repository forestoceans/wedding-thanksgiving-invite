# Feature Spec: RSVP 表单邮件通知

## Overview

当访客填写并提交 RSVP 出席回执表单后，将表单内容通过邮件发送到一组配置好的 QQ 邮箱地址。无需数据库存储，仅做通知转发。

---

## Goals

- 表单提交后，将宾客填写的信息（姓名、出席人数、祝福留言）发送到指定邮箱
- 支持配置多个收件邮箱（列表形式），所有邮箱同时收到通知
- 邮件内容只需包含表单数据即可，不要求美观排版
- 对访客侧体验无影响：成功发送后仍显示现有的"已收到您的回复"成功页面

---

## Non-Goals

- 不需要数据库或后端持久化存储
- 不需要邮件 HTML 模板美化
- 不需要访客收到确认邮件
- 不需要后台管理界面

---

## User Stories

**作为新人（网站所有者）**，当宾客提交 RSVP 表单时，我希望第一时间收到邮件通知，以便了解出席情况。

**作为宾客**，我填写表单后看到成功提示，不感知后台是否发送了邮件。

---

## Functional Requirements

### 表单字段（现有，无需修改）

| 字段 | 类型 | 必填 |
|------|------|------|
| 姓名 | 文本 | 是 |
| 出席人数 | 数字（1–10） | 是 |
| 祝福留言 | 文本 | 否 |

### 邮件发送

- 触发时机：用户点击"确认出席"并通过前端校验后
- 收件人：从配置读取的 QQ 邮箱列表（支持 1 个或多个）
- 邮件主题：例如 `[婚礼回执] 姓名 出席确认`
- 邮件正文：包含所有表单字段的纯文本内容即可
- 所有收件邮箱同时发送（群发，非依次发送）

### 配置方式

- QQ 邮箱列表通过环境变量配置（如 `RECIPIENT_EMAILS=a@qq.com,b@qq.com`）
- 发件方式需调研可行方案（见 Open Questions）

### 错误处理

- 邮件发送失败时：访客侧仍显示成功（不因通知失败影响用户体验）
- 发送失败可在服务端日志中记录，不需要重试机制

---

## Technical Notes

- 当前项目为纯前端（Next.js），发送邮件需要一个服务端接口（API Route）
- 邮件服务可选方案：QQ 邮箱 SMTP、第三方邮件服务（如 Resend、EmailJS）
- 若使用 EmailJS，可在纯前端直接调用，无需 API Route

---

## Decisions

| 项目 | 决定 |
|------|------|
| 发件方式 | QQ 邮箱 SMTP（需在 QQ 邮箱设置中开启 SMTP 并获取授权码） |
| 发件人邮箱 | `oceans-mail@qq.com` |
| SMTP 服务器 | `smtp.qq.com`，端口 `465`（SSL） |

## Environment Variables

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `QQ_SMTP_USER` | 发件人 QQ 邮箱 | `oceans-mail@qq.com` |
| `QQ_SMTP_PASS` | QQ 邮箱 SMTP 授权码（非登录密码） | `xxxxxxxxxxxxxxxx` |
| `RECIPIENT_EMAILS` | 收件人列表，逗号分隔 | `oceans-mail@qq.com,2275545602@qq.com` |

## Acceptance Criteria

- [ ] 宾客提交表单后，所有配置的 QQ 邮箱均收到包含姓名、人数、留言的邮件通知
- [ ] 支持通过环境变量配置多个收件邮箱
- [ ] 邮件发送失败不影响访客侧的成功提示展示
- [ ] 本地开发可通过环境变量切换测试邮箱
